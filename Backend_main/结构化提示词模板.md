# 结构化提示词模板：创建Express后端工程并配置Docker部署

## 任务背景
用户需要在Backend_main目录中创建一个Express后端工程，配置Docker环境，并实现文件上传和下载功能。

## 技术要求
- 使用最新版本的Express
- 使用Node.js 18+版本
- 使用Docker进行容器化部署
- 实现文件上传和下载功能
- 提供GET /getList API，返回包含在线图片URL的列表

## 实现步骤

### 1. 目录创建与项目初始化
- 在FullStack目录下创建Backend_main目录
- 切换到Backend_main目录
- 使用`npm init -y`初始化Node.js项目
- 安装Express依赖：`npm install express`

### 2. 核心功能实现
- 创建`index.js`文件，实现以下功能：
  - 静态文件服务，通过`/downloads`路径提供文件下载
  - 确保downloads目录存在
  - `/getList` API，接受`type`查询参数，返回包含5个项目的列表
  - 列表中的每个项目包含：id、name、type、value、isActive、imageUrl、description、createdAt
  - `imageUrl`字段使用在线图片URL（如https://picsum.photos）
  - 健康检查接口`/health`

### 3. 文件上传功能
- 安装multer依赖：`npm install multer`
- 在`index.js`中添加文件上传功能：
  - 配置multer存储，设置文件存储路径为downloads目录
  - 创建POST `/upload`接口，接受名为"file"的文件字段
  - 上传成功后返回文件信息，包括原始文件名、存储文件名、文件路径、文件大小、MIME类型和下载URL

### 4. Docker配置
- 创建`Dockerfile`文件，使用Node.js 18-alpine作为基础镜像
- 创建`run-docker.ps1`脚本，包含构建和运行Docker容器的命令
- 构建Docker镜像：`docker build -t backend-main .`
- 运行Docker容器：`docker run -d -p 3000:3000 --name backend-main backend-main`

### 5. 功能测试
- 测试GET /getList API：`curl http://localhost:3000/getList?type=test`
- 测试文件上传功能：`curl -X POST -F "file=@./test-upload.txt" http://localhost:3000/upload`
- 测试文件下载功能：`curl http://localhost:3000/downloads/sample.txt`
- 验证Docker容器中的文件：`docker exec -it backend-main ls -la /app/downloads`

### 6. 版本控制
- 创建`.gitignore`文件，忽略不必要的文件和目录，包括`downloads`目录

## 预期结果
- Express后端服务成功运行在Docker容器中
- 容器映射到主机的3000端口
- GET /getList API返回包含在线图片URL的列表
- POST /upload API能够上传文件并返回下载URL
- 静态文件服务能够提供文件下载
- 所有配置文件都已提交到Git仓库
